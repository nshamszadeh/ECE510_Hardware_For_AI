1. I am trying to demonstrate a quantified speedup of the RAVE AI audio synthesis pipeline using a hardware implementation of the audio encoder.
2. Today, RAVE runs entirely in software — either on a CPU or a GPU. CPUs process the audio filterbank and convolutional encoder layers as general-purpose floating point operations, which is flexible but wasteful: these layers have fixed weights after training and a highly regular, repetitive computation pattern that general processors cannot fully exploit. GPU acceleration improves throughput dramatically but draws far more power than the workload requires, making it unsuitable for low-power or embedded deployments. An embedded implementation exists using NVIDIA's Jetson Nano, but this still relies on a general-purpose GPU core rather than dedicated silicon. The fundamental limit is that no implementation targets the specific computational structure of the RAVE encoder in custom hardware — meaning the potential efficiency gains from a purpose-built accelerator remain unrealized.
3. What's new in my approach is the custom hardware accelerator that optimizes the specific computational structure of the RAVE encoder. This approach will work because of the clearly defined computational stages being decoupled in a manner that allows for targetted optimizations. First, decomposing incoming audio into 16 frequency bands can be parallelized and pipelined directly in hardware. Second, the convolutional layers map naturally onto a pipelined MAC architecture where every hardware resource is active every clock cycle. A general purpose CPU or GPU running these computational stages may be bottlenecked by memory access overhead and instruction scheduling costs: this problem is eliminated by the direct hardware implementation.
